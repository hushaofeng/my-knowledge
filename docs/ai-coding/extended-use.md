# AI 编程的扩展应用

> AI 编程远不止写代码和查 Bug。
> 从需求到运维，AI 可以覆盖软件工程全生命周期。

## 二、需求阶段 🔴🔴

### 2.1 需求分析与拆解 🔴🔴

| 能力 | 具体应用 | Prompt 示例 |
|------|---------|-------------|
| 需求解读 | 将模糊需求转化为清晰的功能描述 | "将以下用户故事拆解为可开发的功能点" |
| 需求评审 | 发现需求中的矛盾、遗漏、不合理之处 | "审查以下需求文档，找出潜在的逻辑矛盾和遗漏" |
| 用户故事编写 | 将业务需求转化为用户故事 | "为'用户注册'功能编写用户故事，包含正常和异常场景" |
| 验收标准 | 自动生成验收标准 | "为以下用户故事生成验收标准（Given-When-Then格式）" |

**示例**：产品经理说"用户可以登录"，AI 帮你拆解为：

```
功能点1：登录页面 UI（账号+密码+验证码+登录按钮）
功能点2：登录接口（参数校验、认证、Token 生成）
功能点3：异常处理（账号不存在、密码错误、账号锁定、验证码过期）
功能点4：记住我功能（Token 续期）
功能点5：第三方登录（微信/支付宝 OAuth）
功能点6：安全策略（登录失败次数限制、验证码策略）
```

## 五、测试阶段 🔴🔴🔴

### 5.1 测试用例生成 🔴🔴🔴

| 测试类型 | AI 能做什么 | 价值 |
|---------|-----------|------|
| 单元测试 | 根据函数签名和逻辑生成测试用例 | 提升覆盖率 |
| 集成测试 | 根据接口定义生成集成测试 | 接口质量保障 |
| 边界测试 | 自动生成边界值和异常场景用例 | 发现隐藏 Bug |
| 性能测试 | 生成 JMeter/K6 压测脚本 | 性能基线建立 |
| 安全测试 | 生成安全测试用例（OWASP Top 10） | 安全左移 |

**示例**：AI 生成的测试场景：

```
函数：transfer(fromAccount, toAccount, amount)

正常场景：
- 转账成功，余额正确扣减和增加
- 转账记录正确保存

边界场景：
- 金额 = 0
- 金额 = 账户余额（刚好转完）
- 金额 = 账户余额 + 0.01（余额不足）

异常场景：
- 余额不足
- 账户不存在
- 账户已冻结
- 金额为负数
- 转给自己
- 并发转账（竞态条件）
- 数据库异常（事务回滚）
```

### 7.1 日志分析 🔴🔴

| 能力 | 具体应用 |
|------|---------|
| 日志解析 | 解析非结构化日志，提取关键信息 |
| 异常检测 | 识别异常日志模式 |
| 根因分析 | 根据日志堆栈分析问题根因 |
| 趋势分析 | 分析日志趋势，预警潜在问题 |

**示例**：将错误日志输入 AI：

```
2026-07-08 14:23:01 ERROR [pool-3-thread-2] c.a.service.OrderService - 
  Failed to process order: java.sql.SQLTimeoutException: 
  Query timed out after 30s: SELECT * FROM orders WHERE user_id=? AND status=?

AI 分析：
1. 根因：SQL 查询超时，可能是 user_id+status 组合缺少索引
2. 建议：ALTER TABLE orders ADD INDEX idx_user_status(user_id, status)
3. 临时方案：增加查询超时时间或添加缓存层
4. 关联风险：如果查询量持续增大，可能引发连接池耗尽
```
